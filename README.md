# Energy Constraint Q&A Agent

This project implements a specialized AI agent designed to answer natural language questions about energy market constraints in the MISO (Midcontinent Independent System Operator) market. The agent leverages the Google Agent Development Kit (ADK) and Google's Generative AI models to interact with data stored in Google BigQuery, providing insightful, data-driven answers.

***Under current verion, the agent is actually a BigQuery connecting agent. Therefore, you can ask anything related to BigQuery data, and you can also ask questions about all markets besides MISO.***

## Features

*   **Natural Language Querying:** Ask complex questions about energy market data in plain English.
*   **BigQuery Integration:** The agent can directly query BigQuery datasets to retrieve the information needed to answer your questions.
*   **Dynamic SQL Generation:** Automatically generates and validates SQL queries based on your questions and the available table schemas.
*   **Configurable and Secure:** Uses environment variables to control access to specific BigQuery datasets and tables, ensuring the agent only interacts with the data you permit.
*   **Extensible Framework:** Built on the modular Google ADK, allowing for easy extension with new tools and capabilities.

## How to Run

Follow these steps to set up and run the Energy Constraint Q&A Agent.

The downloading of Google Cloud SDK is probably required along the way.

### 1. Prerequisites

*   Python 3.9+
*   Access to a Google Cloud Platform (GCP) project with BigQuery enabled.
*   A Google API Key with the Generative AI API enabled.

### 2. Clone the Repository

```sh
git clone <repository-url>
cd energy-constraint-qa # Run only if needed
```

### 3. Set Up a Virtual Environment

It is highly recommended to use a virtual environment to manage project dependencies.

```sh
python -m venv venv
source venv/bin/activate  # On Windows, use `venv\Scripts\activate`
```

### 4. Install Dependencies

Install all the required Python packages from the `requirements.txt` file.

```sh
pip install -r requirements.txt
```

### 5. Configure Environment Variables

Create a `.env` file in the root of the `energy-constraint-qa` directory. You can do this by copying an example file if one is provided, or by creating a new file.

```sh
# If you have an example file:
# cp .env.example .env

# Otherwise, create a new file:
# touch .env
```

Now, open the `.env` file and populate it with the necessary values as described in the **Environment Variables** section below.

### 6. Run the Agent

Once your `.env` file is configured with your question and API credentials, you can run the agent with a single command:

```sh
python src/main.py
```

The agent will process the `QUESTION` from your `.env` file and print the final answer to the console.
